# /workflow — インタラクティブワークフロー選択

**カテゴリ**: Orchestration（統括）

実行したい処理を対話的に選択し、段階的に実行するワークフローオーケストレーターです。初めて使うユーザーに最も推奨されるスキルです。

## 使用方法

```bash
/workflow
/workflow ./path/to/source
/workflow ./path/to/source --output=./custom-output/
```

## 概要

`/workflow` を実行すると、まず実行タイプの選択を求められます。

| 選択肢 | 説明 | 実行フェーズ |
|--------|------|-------------|
| **完全パイプライン実行**（推奨） | 調査→設計→コード生成まで全フェーズ | Phase 0〜13 |
| **分析・評価まで実行** | 調査→分析→MMI/DDD評価→統合 | Phase 0〜2.5 |
| **設計から実行** | DDD再設計→ScalarDB設計→実装→コード生成 | Phase 3〜8 |
| **個別フェーズ選択** | サイジング見積もり・ドメインストーリー等を含む全フェーズから選択 | 任意 |

## 前提条件

- 特になし（スタンドアロンで実行可能）
- 対象ソースコードのパスは対話で指定可能

## 実行ステップ

1. **実行タイプの選択** — AskUserQuestion で4つの選択肢を提示
2. **対象パスの確認** — ソースコードの場所を確認
3. **出力先の確認**（オプション） — デフォルト `./reports/` または カスタムパス
4. **選択に基づく実行** — 選ばれたフェーズのスキルを順次呼び出し
5. **進捗管理** — `work/{project}/workflow-progress.json` で進捗追跡
6. **完了後の提案** — 次に実行可能なフェーズを提案
7. **エグゼクティブサマリー生成**（オプション）
8. **HTML総合レポート生成**（オプション） — `/compile-report` の呼び出し

## 出力ファイル

選択したワークフロータイプに応じて異なります。全フェーズ実行の場合は `reports/` 配下の全ディレクトリに出力されます。

進捗追跡ファイル: `work/{project}/workflow-progress.json`
